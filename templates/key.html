ENABLE THE API

Enable API for Project

In order to utilize the Google Custom Search functionality you need to establish access. This is done via the Google Cloud dashboard. With your project selected (if using GAE app) open the 'Product & Services' dropdown from the top left of the screen and select the API Manager link:

In the resulting Overview page search for 'Custom Search API' and select the resulting link:

Next you'll need to click the 'Enable' toggle button to enable the service with your project, and this will bring up a request to establish Credentials which are required to use the API as we intend to with this guide. Click the 'Go to Credentials' button to proceed:

Secure an API Key

At this stage you can setup an API key, Client ID or Service Account with billing/quotas to suit your needs. You can also follow along with the prompted questions to help guide you to the right type of access request. For the purposes of this tutorial we'll setup a free (100 queries per day max) web browser API key. Note the resulting API Key for future use, and you may want to keep it private particularly if you've setup billing and additional access.

We've now successfully enabled our app to use the Google Search API, as well as secured an API key to make HTTP requests to the API from any app. In the next section we'll continue to setup the API's search criteria and fine tune it's usage.
