INTRODUCTION

Google Custom Search

It's the 'Digital Age', where access to near limitless volumes of information has become ubiquitous. Google (parent Alphabet) is now the largest publicly traded company on earth by market capitalization, impressive for a tech company that was founded less then 20 years ago in 1997. It started as an internet search engine, using a powerful new search algorithm, and Google Search continues to this day as a key flagship offering.

With search being such an important tool, including imbedded searches, Google has developed a 'Custom Search API' that allows developers to leverage Google's trusted search functionality within their own applications. The API offers a number of interesting features, many of which will be discussed in this How-To guide, including the ability to do full text searches and narrow search results to a variety of target domains from top level domains, down to individual hosts or even directories. This How-To guide will take a beginners approach to setting up the API and utilizing it within your own apps. The guide will focus on the obvious target of Google App Engine, but can also be generalized to other Cloud or Mobile offerings.

When first attempting to use this tool I found the setup confusing, and while some documentation exists it was hard to find, targeted at more advanced users and not centralized. The goal of this guide is to consolidate this information and provide a single source beginner's walkthrough of setting up and using the Google Search API on a Google App Engine hosted webpage. This site will provide an overview of the process of establishing access to the API, setting up specialized search criteria, the API interface & response and provide an example of the API in action. Code snippets will be provided throughout the guide, but the source code for this site (including the API interaction from the example page) can be downloaded and reviewed from GitHub here.
